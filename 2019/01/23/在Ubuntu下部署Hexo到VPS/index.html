在Ubuntu下部署Hexo到VPS

2019-01-23

把之前在Windows上部署Hexo的文章重新写一遍，这次系统换成了Ubuntu，对于整个部署流程也有了一个更清晰的了解，加上VPS那边无需改动，整个操作一气呵成

  • 本地环境：Ubuntu 18.04 LTS
  • VPS：CentOS7（搬瓦工+万网域名）

本地部署Hexo

Hexo是一个快速、简洁且高效的博客框架。Hexo使用Markdown(或其他渲染引擎)解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

Git

Git是一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理。

通过终端进行安装
安装Git： sudo apt install git
更新Git：
add-apt-repository ppa:git-core/ppa （添加PPA源）
sudo apt update
sudo apt install git
查看Git版本：git --version
使用GIt之前，需要输入以下信息：
git config --global user.name+空格+你的名字
git config --global user.email+空格+你的邮箱

Node.js

Node.js 是一个基于Chrome JavaScript 运行时建立的一个平台。 Node.js是一个事件驱动I/O服务端JavaScript环境，基于Google的V8引擎，V8引擎执行Javascript的速度非常快，性能非常好。

先安装npm (NPM是随同NodeJS一起安装的包管理工具，能解决NodeJS代码部署上的很多问题)
sudo apt install npm
安装Node.js：sudo apt install Nodejs
更新Node.js：
curl -sL https://deb.nodesource.com/setup_9.x | sudo -E bash -
sudo apt-get install -y nodejs
查看Node.js版本：node -v

配置Hexo

建立一个Hexo专用文件夹：mkdir hexo（所在路径：/home/hexo）
打开文件夹：cd hexo
通过npm安装Hexo：npm install -g hexo-cli
初始化：Hexo int
至此本地部署工作完成，顺便测试下是否成功
输入hexo server

浏览器输入 http://localhost:4000 即可访问默认主页

VPS部署Hexo

在Ubuntu上用终端Terminal和自带的文件管理器Nautilus就可以直连服务器，SSH和sftp管理操作一步到位，非常舒服√

Nginx

Nginx是一个网页服务器，它能反向代理HTTP, HTTPS, SMTP, POP3, IMAP的协议链接，以及一个负载均衡器和一个HTTP缓存。相较于Apache、lighttpd，Nginx具有占有内存少，稳定性高等优势。

添加CentOS 7 Nginx yum资源库：sudo rpm -Uvh http://nginx.org/packages/centos/7/noarch/RPMS/nginx-release-centos-7-0.el7.ngx.noarch.rpm
安装Nginx：yum install -y nginx
启动nginx：systemctl start nginx.service
打开域名IP，显示Welcome to Nginx

配置Nginx
修改/etc/nginx/conf.d/default.conf的server_name，保存退出
查看配置是否有错误：nginx -t

Git

CentOS7中虽然自带Git，不过依然需要更新，过程有点麻烦
先安装Git的依赖包：

yum install curl-devel expat-devel gettext-devel openssl-devel zlib-devel gcc perl-ExtUtils-MakeMaker
删除已有的Git：yum remove git
下载&安装Git源码：

查看当前版本
git --version
记得使用前输入个人信息
git config --global user.name+空格+你的名字
git config --global user.email+空格+你的邮箱

创建Git用户

创建用户：adduser git
设置密码：passwd git
改变权限值用于读写：chmod 740 /etc/sudoers
赋予sudo权限：vi /etc/sudoers
添加：
root ALL=(ALL:ALL) ALL
git ALL=(ALL:ALL) ALL
修改回文件权限：chmod 440 /etc/sudoers

配置ssh登陆

生成密钥：ssh-keygen -t rsa
直接回车会在~/.ssh目录

密钥下载到本地，公钥改名

设置authorized_keys文件的权限
cd .ssh
cp id_rsa.pub authorized_keys
chmod 700 authorized_keys
修改sshd_config：
vim /etc/ssh/sshd_config
找到下面代码前的#并删除

重启ssh服务：systemctl restart sshd.service

切换git用户：su git
切换到git用户目录： cd /home/git
创建.ssh文件夹：mkdir ~/.ssh
创建authorized_keys文件并把刚才下载密钥内容粘贴进去
vim ~/.ssh/authorized_keys
修改以下两处的权限
chmod 600 ~/.ssh/authorzied_keys
chmod 700 ~/.ssh

这时候可以在本地通过终端ssh git@vps地址测试能否登陆

创建git裸库

回到git目录：cd /home/git
使用git用户创建git裸仓库：git init --bare blog.git
创建网站目录：
cd /var/www
mkdir blog
chown git:git -R /var/www/blog
检查三个权限：
ll -a /var/www/
ll -a /home/git/

配置git-hooks

su git
cd /home/git/blog.git/hooks

赋予post-receive执行权限
chmod +x post-receive
最后将git目录所有权设置为git。
cd /home/git
chown -R git:git hexo.git

至此服务器端的配置结束

实现自动化部署

回到本地端，修改Hexo目录下的_config.yml

安装git的push插件：npm install hexo-deployer-git --save
最后将本地目录生成静态网页并推送到VPS
hexo g && hexo d
完事

总结流程坑点

1.没搞清本地的hexo安装路径
2.没搞清远程的hexo安装路径
3.nginx的default.conf配置问题

4.hooks里的post-receive配置问题
